设计师周报助手

2026-03-16 至 2026-03-21 周报。依据 2026-03-20 Solvely 设计周会发言整理。标题在外，正文在下方卡片中。右侧图标仅复制正文。

工作重心与用户价值

本周工作重心：设计周会发言里聚焦商业化、解题可视化和设计协作效率

1. 在周会上汇报了周包价格实验推全后的正向结果，明确当前价格策略效果良好，转化率和入金率都有提升，并且已经继续加测新实验组。

2. 推进 AIO 和 Solver 权益提示样式与交互统一，把标题、正文和主按钮等结构拆清，目标是把不同场景下的权益提示口径收拢到同一套设计语言。

3. 围绕 Solver 解题可视化继续推进方案收敛，周会上明确讲到实时 SVG 动图讲解效果不够理想，因此转向更稳定的视频化方案，优先保证用户看到的讲解质量。

4. 高亮引用实验已经跑出正向信号，留存率和入金率都有提升，本周继续进入二期实验，重点验证强制触发和点击触发两种曝光方式的差异。

市场 / 竞品 / 用户洞察

1. 竞品调研层面，周会上提到 Turbo 会在 loading 过程中持续给用户提示，Studyfetch 上传入口丰富但理解成本偏高，Gizmo 通过更明显的上传入口降低门槛，Sally 用视频引导但存在卡顿问题。

2. 由此反推到自身产品，当前最核心的问题不是功能不够多，而是 sample 入口偏弱、案例单一、上传弹窗没有完整表达用户能获得什么，价值呈现链路还有明显优化空间。

3. 数据侧的判断也更明确了，真正值得持续放大的不是单次点击，而是曝光后是否带来留存和付费结果，因此高亮引用实验会继续以“结果指标”而不是“表层点击”作为判断标准。

4. 解题讲解方案的洞察也比较直接：在教育场景里，技术方案是否先进不重要，重要的是用户能否稳定看懂，因此 SVG 路线让位给更可交付的视频方案是必要取舍。

AI 应用案例

1. 周会里明确分享了 AI 工具使用判断：设计协作优先多用 Codex 和 Cursor，少依赖不可控、没有 GUI 的纯命令式链路，核心原因是可视化和可回退能力更适合设计工作流。

2. 本周还展示了用 Cursor 结合 Figma 插件去创建 variable、style 和 component 的尝试，这条链路已经能帮助把一部分手工搭建设计系统的工作自动化。

3. 在知识管理层面，也把 AI 工作方式进一步结构化：通过长期记忆、goals、knowledge 和 task 的分层，让同一个 agent 可以围绕同一套项目资料持续工作，而不是每次重讲背景。

4. 从设计落地角度看，本周最有价值的 AI 不是“生成一个炫的效果”，而是把设计资产迁移、界面复刻、资料读取和协作沟通这些重复劳动压缩掉。

自驱推动事项

1. 在周会上主动把设计侧的数据获取问题摊开来讲，推动后续通过统一文档管理、放开 PRD 可见范围和引入飞书 MCP 辅助取数，减少设计师拿不到数据、拿到也读不懂的情况。

2. 主动输出了对 Cursor、Codex 和 Figma 协作方式的使用建议，不只停留在个人试用，而是已经准备把开源免费工具和方法发到群里，帮助团队一起提升效率。

3. 在解题可视化方向上没有停留在“做出一个 demo”，而是及时在周会上同步路线修正，从 SVG 切到视频化方案，避免团队在效果不佳的技术路径上继续投入。

4. 把价格实验、高亮引用和权益提示这些分散事项重新组织成一条更清晰的汇报主线，让周会上的讨论能围绕“结果如何、为什么有效、接下来怎么放大”继续推进。
