工作重心与用户价值

Onboarding 学科专业化凸显

PRD 的核心背景是：商科与医科用户的价值诉求差异已经大到不能再用同一套 onboarding 话术覆盖。商科用户更在意能不能快速产出可交付内容和决策洞察，医科用户更在意记忆效率、复杂结构理解和内容权威性。如果还用统一入口去讲价值，前几屏就会把关键动机讲偏。

基于这个判断，这次把流程重写为“通用信任建立 - 学科分流 - 学科价值强化 - 准确性顾虑处理”，在用户完成学科选择后再进入对应价值页，把抽象能力翻译成该学科能立即感知的收益，而不是继续先展示泛化能力再让用户自己理解。

产出上已经形成完整流程、分学科价值页、权益承接页和埋点方案，并明确了 AB 分组。这项设计的实际价值，是提高新用户在 onboarding 阶段的价值感知速度，减少“能做很多，但跟我有什么关系”带来的早期流失。

实时生成 HTML 视频讲解

PRD 的背景判断很明确：Gemini 3.0 Pro 已经把“按题目实时生成讲题视频”的技术门槛降下来了，但美国 Gen Z 用户的注意力窗口很短，直接做成 Gauth 那类长链路语音教学不一定成立。这个能力要成立，前提是入口足够轻，反馈足够快，不能让用户先承受冗长等待和过重心智。

所以这次不是简单增加一个视频按钮，而是把整条体验链路补齐成可验证闭环：答案页入口、生成中状态、生成成功/失败、全屏横屏播放、视频反馈与成功通知。重点是把“视频讲解”从概念展示改成真正可观察点击率、播放完成率和入金变化的实验能力。

这项工作的价值在于，先用最小可用体验验证视频是否值得继续深挖，而不是先投入重资产把它做成复杂教学系统。

多模态追问输入功能

PRD 的分析指出，用户追问最集中的几类需求其实不是“随便聊聊”，而是补图、补上下文、输入公式、展开解释和重解。原有纯 chatbox 方案对公式输入不友好，也很难让用户在追问时同时参照原题和原答案，导致提问不够准、模型回复也容易漂。

基于这个问题，这次重构了追问栏默认态、Explain further / Simplify 快捷路径、Try again 切模式、追问页消息操作、发送失败重试，并补齐了 Phase 1 的分屏弹层方案。方向上不再把追问当独立聊天页，而是把它做成围绕当前解题内容展开的学习操作区。

这项设计的意义是降低追问门槛，让用户更容易问到点上，也让模型更容易基于当前答案给出针对性回复，从而提升追问的实际使用率和有效性。

Solution 页信息结构重构 2.1 / 多模型导航收敛

PRD 里定义的问题非常具体：当前答案页把题目图、多模型切换、Thinking、Final Answer、步骤、视频、追问等模块全部堆叠在一起，用户注意力被拆散；同时 2.1 期还要解决一个更实际的问题，用户从 Fast 切到其他模式后无法回看 Fast 答案，等待新答案时只能空等，体验不连续。

因此这次的设计不是单做一个模式菜单，而是围绕“信息结构和切换逻辑”一起重构：把页面整理成更明确的竖向瀑布流和吸顶横向 Tab，再用顶部模式切换入口承接多模型切换。细节上补齐了已解模型与未解模型的点击规则、加载中状态、气泡引导、Solved with / More to try 的动态展示逻辑，以及等待中仅允许查看已解模式的约束。

这项工作的核心价值，是把多模型体验从“切一下再等”改成“切换过程中也能回看和比较”，减少等待空档，提高答案页的可读性和连续性。

多页拍引导及商业化策略

PRD 的背景是：用户真实拍题场景已经明显多页化，尤其商科题目经常跨页出现题干、表格、图和小问。只拍一页会直接造成信息缺失，模型后面再怎么解释都救不回来。多页拍本质上是在输入阶段提前消灭错误，而不是事后修补答案。

另一个关键判断是，多页拍不是用户天然会发现的能力，必须靠引导才能变成习惯动作。数据也支持这一点：7 天内 40% 用户只用 1 次，但进入前 20% 后频次会直接跳到 6 次，说明它不是普遍高频，却对高价值用户有明显粘性。

基于此，这次把多页拍设计成“先教育、后承接”的路径，补齐首次进入裁剪页的引导气泡、Add more 提示、免费使用次数门槛和后续付费承接。它的价值不只是多一个功能点，而是减少因输入缺页导致的错误答案，同时把高价值能力更稳定地转成商业化入口。

价格测试与策略承接对齐

价格策略这条的背景不是单纯做运营配置，而是已有数据表明：大量试用用户会很快取消，且取消前往往已经完成核心解题需求，说明“试用”对部分人群并不一定构成有效转化。2 月这轮策略里，大学生无试用组已经验证出入金提升 25%，因此需要把局部实验推进成更大范围的承接统一。

这次工作的重点在设计侧不是改价格本身，而是把 Onboarding Intro、Onboarding Shop、Normal Shop 和弹窗页的展示逻辑、价格文案和入口口径统一起来，避免策略在后台切了，但用户在不同入口看到的是不同说法，最终把实验收益抵消掉。

这项工作的价值，是让价格实验真正落到用户可感知的一致体验上，减少入口间信息错位造成的理解偏差和转化损耗。

整体结果与用户价值

这一阶段真正推动的是把多条核心链路做成可验证、可实验、可交付的设计系统，而不是单点页面优化。结果上，已经形成多条需求的完整交互链路、分学科价值表达、模式切换规则、追问分屏逻辑、多页拍承接逻辑和价格策略承接口径。

对用户和业务的直接价值，是减少输入缺失、等待空窗和理解断层，提高用户对能力的感知效率，并为后续转化实验提供更稳定的设计前提。

市场 / 竞品 / 用户洞察

这段时间最明显的新增认知有两条。第一，学科差异对价值表达的影响比原来判断更大。Onboarding 文档里已经明确：商科用户更关注可交付内容和决策洞察，医科用户更关注记忆效率、复杂结构理解和内容权威性，用一套统一话术很难同时打中两类核心动机。

第二，多页拍的用户行为出现明显两极分化。大部分用户只是浅尝，但一旦跨过前几次使用门槛，频次会明显跃迁，说明这类能力一旦被教育成功，会变成高粘性入口。

我的判断是，当前产品更应该优先强化“高价值能力的识别与教育”，而不是继续堆叠泛化能力展示。建议后续继续推进按学科表达和多页拍引导，但评估不能只看点击，要重点看进入深层链路后的留存、追问和付费表现。

AI 应用案例

这段时间最值得沉淀的 AI 案例，是把“视频讲题”从静态概念推进成可运行的产品形态。原来答案页的解释主要停留在文字与步骤展示，用户是否愿意继续理解、是否能看懂，更多依赖阅读耐心。

现在借助 Gemini 3.0 Pro 的能力和约束式 prompt，把题目讲解转成可实时生成的 HTML 视频方案，设计不再只停留在“入口长什么样”，而是能围绕生成中、生成成功、播放、反馈这整条真实链路设计。

变化不只是形式更丰富，而是让 AI 讲解从抽象能力变成可以被验证点击率、播放完成率和转化贡献的实验对象。这个方法可团队推广，但前提是继续保持“先做最小闭环、再扩展体验”的节奏。

自驱推动事项

这段时间主动推进的重点，是把多个需求里的“承接链路”和“状态规则”做统一收口，而不是各自孤立推进。你在模型菜单切换里主动补充了已解模型与当前模型的展示规则、置灰规则和状态 icon 回退；在 Solution 页 2.1 里把多模型切换的逻辑从单次交互扩展到“等待中也能回看”。

之所以值得做，是因为这些问题单看都是小细节，但如果不提前统一，最后会在评审、研发和上线阶段反复返工。

当前阶段已经完成规则层面的收口，并沉淀到了 PRD 或你自己的工作文档里。下一步更值得继续做的是，把状态规则、承接规则、置灰规则抽成可复用模板，减少每个需求都重新定义一次。
